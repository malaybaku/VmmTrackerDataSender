# Web UIブラッシュアップ - タスクリスト

## 目的

モバイルでのユーザビリティ向上、トラッキングステータス表示の改善、日英対応を行う。

要件の詳細: `docs_ignored/Requirement_WebBrushup.md`

---

## 設計方針

- レイアウト・UI改善を先に行い、文言を確定させてから日英対応を実施する
- 日英対応は文言定義ファイル(ja/en)で管理し、ブラウザ言語で出し分ける
- プライバシーポリシー・ライセンスページは日英対応の対象外
- 既存のCSS変数やカラーパレットを整理し、明るめの配色に統一する

## Phase A: レイアウト・配色調整

### A-1: ヘッダーエリアの新設と接続ステータスの移動

**現状**: WebRTC接続ステータス(`#connection-status`)とDataChannelステータス(`#status`)がコントロールバー内の`<select>`の右に横並びで表示されている。

**変更内容**:
- 画面上部にヘッダーエリアを新設する
- WebRTC接続に関するステータス表示をヘッダーエリアに移動する
- カメラのトラッキング状況はヘッダーには表示しない（Phase Bで別途対応）
- DataChannelの状態表示(`#status`)もヘッダーに移すか、用途を再検討する

**対象ファイル**: `index.html`, `app.css`, `main.ts`, `ui.ts`

### A-2: コントロールバー・フッターのモバイル対応

**現状**: Androidの縦画面でコントロールバーやフッターがナビゲーションバーに被る。`#app`が`height: 100vh`で`overflow: hidden`のため、フッターがスクロールできない領域に隠れる場合がある。

**変更内容**:
- `100vh`の代わりに`100dvh`(dynamic viewport height)を使用し、モバイルのナビゲーションバー・アドレスバーを考慮した高さにする
- コントロールバーとフッターが常に画面内に収まるようレイアウトを調整する
- スクロールが不要な状態を目指しつつ、万一収まらない場合はスクロール可能にする

**対象ファイル**: `app.css`, `index.html`

### A-3: プレビューエリア・背景の配色見直し

**現状**: 映像エリア背景が`#000`(黒)、コントロールバーが`#2a2a2a`、フッターが`#1a1a1a`。全体的に暗い。

**変更内容**:
- 映像エリアの背景をもう少し明るい色（ダークグレー系）に変更する
- コントロールバー、フッター等の背景色も合わせて調整する
- 必要に応じて文字色に黒色系を採用してもよい（要件に明記あり）
- CSS変数でカラーパレットを一元管理できるようにする

**対象ファイル**: `app.css`, `common.css`

### A-4: 動作確認（レイアウト）

- PCブラウザ（デスクトップ幅）で崩れがないことを確認
- モバイル相当の幅（DevToolsのレスポンシブモード等）で確認
- ヘッダー・コントロールバー・フッターがすべて画面内に収まることを確認

## Phase B: トラッキングステータス表示の改善

### B-1: TrackingStatusの状態整理とカメラ権限エラー対応

**現状**: `TrackingStatus`は3状態(`TrackingSuccess`, `TrackingNoFace`, `NotTracking`)。`NotTracking`はカメラ権限なしなど限定的なケースで発生。

**変更内容**:
- `NotTracking`状態を廃止し、デフォルトを`TrackingNoFace`(未検出)相当にする
- カメラの利用権限が得られなかった場合は`alert()`等でユーザーに通知する
- `previewRenderer.ts`の`getStatusInfo()`と`TrackingStatus` enumを更新する

**対象ファイル**: `src/types.ts`, `src/previewRenderer.ts`, `src/mediapipe.ts`, `src/main.ts`

### B-2: ステータス表示の色味・デザイン調整

**現状**: 未検出時は`⚠️`アイコン + 黄色(`#ffc107`)で警告色。成功時は`✅`+緑色。テキスト全体が色付き。

**変更内容**:
- 未検出時: アイコン・テキストともにライトグレーなど控えめな色味にする
- 成功時: チェックマークのみ緑色(`#28a745`)、文字は白色(`#fff`)にする
- ステータス表示パネルのUI幅を固定または`min-width`を設定し、状態切り替え時のガクガクを防止する

**対象ファイル**: `src/previewRenderer.ts`, `app.css`

### B-3: pos/rot表示の折りたたみ

**現状**: トラッキングデータ（位置・姿勢の数値）がステータスと一緒に常時表示されている。

**変更内容**:
- デフォルトではステータス表示（「顔未検出」/「トラッキング成功」）のみを表示する
- pos/rot数値の表示エリアは折りたたみ可能にし、タップで展開/非展開を切り替えられるようにする
- 折りたたみ状態は画面遷移しない限り維持する（永続化は不要）

**対象ファイル**: `src/previewRenderer.ts`, `app.css`

### B-4: 動作確認（ステータス表示）

- トラッキング成功→未検出→成功の切り替えでUIの幅が変動しないことを確認
- 折りたたみの開閉が正しく動作することを確認
- カメラ権限拒否時にalertが表示されることを確認

## Phase C: トラッキング再開ボタン

### C-1: 再開ボタンの追加と再起動処理の実装

**現状**: 一度トラッキングが停止した場合の再開手段がない。

**変更内容**:
- Preview表示選択の近くに「トラッキングを再開」ボタンを配置する
- タップで`MediaPipeManager`のトラッキング処理を再起動する
  - カメラストリームの再取得→MediaPipeの再初期化、またはMediaPipeのみ再起動（実装時に適切な方法を選択）
- 将来的に不要になる可能性があるため、追加・削除が容易な構造にする

**対象ファイル**: `index.html`, `app.css`, `src/main.ts`, `src/mediapipe.ts`

## Phase D: 日英対応

### D-1: 多言語対応基盤の構築

**変更内容**:
- ブラウザの`navigator.language`を判定し、`ja`の場合のみ日本語、それ以外は英語をデフォルトとする関数を実装する
- 文言定義ファイル（例: `src/i18n/messages.ts`）にja/enのテキストをまとめて定義する
- テキスト取得用のヘルパー関数（例: `t('key')`）を用意する

**対象ファイル**: 新規 `src/i18n/` ディレクトリ

### D-2: 既存文言の日英置き換え

**変更内容**:
- `index.html`内のハードコードされた日本語テキストをD-1の仕組みで動的に設定するよう変更
  - select要素のオプション（表示なし/ランドマーク/映像）
  - 接続モーダルのテキスト
  - その他UIラベル
- `main.ts`内のステータスメッセージ、エラーメッセージ
- `previewRenderer.ts`内のトラッキングステータス文言、データラベル
- フッターのリンクテキスト（Source, Licenses, Privacy Policy）
- **対象外**: `privacy-policy.html`, `licenses.html` の内容

**対象ファイル**: `index.html`, `src/main.ts`, `src/previewRenderer.ts`, `src/ui.ts`, `src/i18n/messages.ts`

### D-3: 動作確認（日英対応）

- ブラウザの言語設定を日本語にした場合に日本語が表示されることを確認
- ブラウザの言語設定を英語にした場合に英語が表示されることを確認
- 定義漏れのテキストがないことを目視確認
